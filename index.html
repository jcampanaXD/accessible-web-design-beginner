Accessible Web Design

What does accessibility mean in the context of the web?

What are POUR Principles?

Perceivable

For content to be perceivable, it must be presented in ways that users can recognize and understand regardless of the sensory abilities they do or do not possess.

Example problems

  • A deaf or hard-of-hearing user trying to watch a video
  • A visually impaired user who cannot interpret an image
  • A blind user who is unable to read on-screen text

Potential solutions

  • Provide captions and transcripts for videos
  • Use descriptive alternative text for images
  • Ensure content is compatible with screen readers

Operable

Something is operable if the user interface and navigation components can be used by everyone.

Example problems

  • A user with a motor disability that prevents them from using a mouse
  • Auto-scrolling content moves too quickly for users to read or interact with
  • A user with epilepsy opens up a webpage with quick, flashing lights

Potential solutions

  • Make all functionality available from a keyboard
  • Give users enough time to read and use content
  • Do not include content that causes seizures

Understandable

A website is considered to be 'understandable' when its content and functionalities are clear to all users.

Example problems

  • Navigation links move unexpectedly when hovered over
  • A form uses jargon that's confusing to some users
  • Error messages provide no guidance on how to fix the problem

Potential solutions

  • Design consistent navigation and layout
  • Use appropriately targeted language and explain technical terms-- aim for an 8th- or 9th-grade reading level when writing for the general public
  • Offer helpful, specific error messages with suggestions

Robust

A 'robust' website supports and can be accessed by many types of devices, including assistive technologies.

Example problems

  • The website breaks in screen readers
  • A form doesn't work in older browsers
  • The website doesn't work properly if someone's browser doesn't run JavaScript

Potential solutions

  • Use valid, semantic HTML that conforms to standards
  • Test compatibility across different browsers and devices
  • Make sure important parts of the site work even if someone's browser doesn't support extra features like JavaScript
